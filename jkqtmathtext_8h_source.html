44/*! \brief this class parses a LaTeX string and can then draw the contained text/equation onto a <a href="http://doc.qt.io/qt-5/qpainter.html">QPainter</a>
45 \ingroup jkqtmathtext
46
47
48 JKQTMathText is a self-contained LaTeX-renderer for Qt. It is used to renderer
49 labels in JKQTPlotter/JKQTBasePlotter, but can be used independently.
50 The class does not depend on any library, except Qt.
51 In particular it actually parses a LaTeX string and draws it in pure C++. It does NOT rely
52 on an installed LaTeX for the rendering!
53
54 \section JKQTMathTextUsage Usage
55 \subsection JKQTMathTextUsageDirect Direct Usage
56 This small piece of C++ code may serve as an example of the usage and capabilities of the class:
57 \code
58 // create a JKQTMathText object.
59 JKQTMathText mathText;
60
61 // configure its properties to influence the rendering (e.g. fonts to use, font size, ...)
62 mathText.useXITS();
63 mathText.setFontSize(20);
64
65 // parse some LaTeX code (the Schroedinger's equation)
66 mathText.parse("$\\left[-\\frac{\\hbar^2}{2m}\\frac{\\partial^2}{\\partial x^2}+V(x)\\right]\\Psi(x)=\\mathrm{i}\\hbar\\frac{\\partial}{\\partial t}\\Psi(x)$");
67
68 // use the draw() methods to draw the equation using a QPainter (here onto a QPixmap)
69 QPainter painter;
70 QPixmap pix(600,400);
71 painter.begin(&pix);
72 mathText.draw(painter, Qt::AlignCenter, QRectF(0,0,pix.width(), pix.height()), false);
73 painter.end();
74 \endcode
75
76 \subsection JKQTMathTextSizing Determining the size of an equation
77
78 In addition there are also functions that allow to calculate the size of the equation, before drawing it (just like the functions in <a href="http://doc.qt.io/qt-5/qfontmetrics.html">QFontMetrics</a> and <a href="http://doc.qt.io/qt-5/qfontmetricsf.html">QFontMetricsF</a>):
79 - getSizeDetail()
80 - getSize()
81 - getAscent(), getDescent()

84 \subsection JKQTMathTextErrorHandling Error Handling
85
86 The class is designed to be as robust as possible and will still return some output, even if the equation contains some errors.
87 Nevertheless, several errors are detected while parsing. You can get a list of error messages using getErrorList() after calling parse().
88 Also parse() will return \c false if an error occured while parsing.

115 - \c \\script{...} \c \\textscript{...} \c \\mathscript{...} : draw the contained text in a script font face \image html jkqtmathtext_fonts.png

140 \subsection JKQTMathTextSuppoertedLaTeXBraces Braces ...
141 Braces in math mode are adjusted in size, so they are a small bit (factor \c brace_factor ) larger than the contents.
142 To enable this you have to write braces with \c \\left and \c \\right. These types of braces are defined (slight
143 differences to LaTeX standard):
144 - \c \\left( \c \\right) : default meaning (), \image html jkqtmathtext_brace_round.png
145 - \c \\left[ \c \\right] : default meaning [], \image html jkqtmathtext_brace_rect.png
146 - \c \\left\\{ \c \\right\\} : default meaning {}, \image html jkqtmathtext_brace_curly.png
147 - \c \\left< \c \\right> : "averaging" braces , \image html jkqtmathtext_brace_tri.png
148 - \c \c \\left\\lfloor \c \\right\\rfloor : floor braces , \image html jkqtmathtext_brace_floor.png
149 - \c \\left~ \c \\right~ \c \\left\\lceil \c \\right\\rceil : ceil braces , \image html jkqtmathtext_brace_ceil.png
150 - \c \\left| \c \\right| : absolute value braces | |, \image html jkqtmathtext_brace_oneline.png
151 - \code \left\| \right\| \endcode : norm braces || ||, \image html jkqtmathtext_brace_dblline.png
152 - You can use \c \\left. or \c \\right. to have only right or only left brace
153 .

178 \section JKQTMathTextSuppoertedFonts Font Handling
179
180 Several fonts are defined as properties to the class:
181 - A "roman" font used as the standard font ( setFontRoman() )
182 - A "sans-serif" font which may be activated with \c \\sf ... ( setFontSans() )
183 - A "typewriter" font which may be activated with \c \\tt ... ( setFontTypewriter() )
184 - A "script" font which may be activated with \c \\script ... ( setFontScript() )
185 - A greek font which is used to display greek letters \c \\alpha ... ( setSymbolfontGreek() )
186 - A symbol font used to display special (math) symbols. ( setSymbolfontSymbol() )
187 - A "roman" font used as the standard font in math mode ( setFontMathRoman() )
188 - A "sans-serif" used as sans serif font in math mode ( setFontMathSans() )
189 - A "blackboard" font used to display double stroked characters ( setFontBlackboard() )
190 - A "caligraphic" font used to display caligraphic characters ( setFontCaligraphic() )
191 .
192
193 These fonts are generic font classes, which font is actually used can be configured in JKQTMathText class with the \c set...() functions mentioned above. You can also use these functions to set the fonts used for math rendering in math-mode:
194 - useSTIX() use the STIX fonts from <a href="https://www.stixfonts.org/">https://www.stixfonts.org/</a> in math-mode<br>\image html jkqtmathparser_stix.png
195 - useXITS() use the XITS fonts from <a href="https://github.com/alif-type/xits">https://github.com/alif-type/xits</a> in math-mode. These are included by default in this library and also activated by default.<br>\image html jkqtmathparser_xits.png
196 - useASANA() use the ASANA fonts from <a href="https://ctan.org/tex-archive/fonts/Asana-Math/">https://ctan.org/tex-archive/fonts/Asana-Math/</a> in math-mode<br>\image html jkqtmathparser_asana.png
197 - useAnyUnicode() use generic Unicode fonts, e.g. "Arial" and "Times New Roman" in math-mode. You should use fonts that contain as many of the mathematical symbols as possible to ensure good rendering results.<br>using "Times New Roman": \image html jkqtmathparser_timesnewroman.png
198 <br>using "Arial": \image html jkqtmathparser_arial.png
199 <br>using "Courier New": \image html jkqtmathparser_couriernew.png
200 <br>using "Comic Sans MS": \image html jkqtmathparser_comicsans.png
201 .
202
203
204 Math-mode is activated by enclosing your equation in \c $...$ or \c \\[...\\] . This mode is optimized for mathematical equations. Here is an example of the difference:
205 - <b>math-mode (XITS fonts are used, whitespaces are mostly not drawn directly, symbol spacing is different)</b> \c $...$: <br>\image html jkqtmathparser_schreq_mathmode.png
206 - <b>normal mode (Times new Roman is used, whitespaces are evaluated directly)</b>: <br>\image html jkqtmathparser_schreq_normalmode.png
